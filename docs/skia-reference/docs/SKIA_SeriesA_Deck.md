# SKIA Series A Technical Narrative

## Current architecture
- One canonical backend runtime with mounted modular systems.
- Multi-client distribution across web, mobile, and desktop shell.
- Voice and policy-governed workflows integrated into backend operations.

## Maturity framing
- Active systems: backend orchestration, policy controls, billing/credits, core client experiences.
- Partial systems: modules that are integrated but still operationally maturing.
- Stub systems: scaffolds explicitly documented as non-production.

## Investor-safe messaging
- Prefer measurable deployment reality over aspirational terminology.
- Keep roadmap claims linked to concrete implementation milestones.
