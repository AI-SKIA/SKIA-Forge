# SKIA Business Deck Blueprints (Engineering-Aligned)

## Positioning guardrails
- Represent only code-backed capabilities as current-state.
- Label roadmap systems as `future-state` with explicit dependency notes.

## Technical claims allowed
- Primary backend runtime and mounted module ecosystem.
- Multi-client delivery (web, mobile, desktop shell).
- Security/policy, billing/credits, and voice service integration.

## Claims requiring caution
- Fully autonomous OS behavior across all subsystems.
- Production readiness for scaffold services or non-wired modules.

## Recommended structure
1. Current architecture baseline
2. Active capability map
3. Gaps and maturation plan
4. Near-term delivery roadmap
