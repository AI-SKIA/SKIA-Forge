# .skia Runtime Metadata Reference

## Core Files

- `.skia/runtime-state.json`: provider and governance runtime snapshots
- `.skia/architecture-baseline-v1.json`: architecture baseline graph state
- `.skia/agent-log.json`: agent execution trail

## Usage Rules

- Runtime metadata should be machine-written, human-readable JSON.
- Operational docs should reference these files as evidence sources, not re-state values manually.

## Consistency

- Keep filenames and references synchronized across docs and indexes.
