# SKIA-Forge Pricing and Packages

This document defines a practical commercial packaging model for SKIA-Forge.

## Product framing

Package SKIA-Forge as governance/orchestration control plane value layered on top of `Skia-FULL` runtime delivery and complemented by `Skia-Status` transparency operations.

## Package 1: Developer

Best for solo engineers and small experiments.

- Core Forge runtime
- Basic orchestration and governance modes
- Standard API access
- Community support

## Package 2: Team

Best for engineering teams adopting AI workflows.

- Everything in Developer
- Team policy presets
- Shared operational dashboards
- Priority support response
- Collaboration-ready governance defaults

## Package 3: Enterprise

Best for organizations with compliance and operational requirements.

- Everything in Team
- Enterprise policy controls and hardening guidance
- Audit/export-ready operational reporting
- Deployment architecture support
- Defined support SLAs and escalation path

## Optional Add-ons

- Custom policy packs
- Integration acceleration services
- Architecture and security review workshops
- Rollout training for platform and security teams

## Commercial framing guidance

- Sell outcomes, not just features:
  - safer velocity
  - reduced rollback risk
  - stronger auditability
- Position Forge as the governance/control multiplier for existing AI development investments.
