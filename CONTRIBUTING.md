# Contributing

## Workflow

1. Create a feature branch from `main`.
2. Keep changes focused and reviewable.
3. Run lint/build checks before opening a PR.
4. Include a clear description of why the change is needed.

## Commit Guidance

- Use concise, action-oriented commit messages.
- Avoid bundling unrelated changes in one commit.

## Pull Requests

- Add context, screenshots (if UI), and test notes.
- Link related issues or docs when available.
- Ensure no secrets are committed.

## Code Quality

- Preserve existing architecture and naming patterns.
- Prefer small, explicit changes over broad refactors unless requested.
